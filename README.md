# 💰 포인트 관리 서비스 - TDD 과제

Spring Boot 기반의 포인트 관리 서비스입니다.  
포인트 충전, 사용, 조회 및 이력 조회 기능을 테스트 주도 개발(TDD) 방식으로 구현하였습니다.

---

## 요구 사항 정의

- 포인트 조회
    - 특정 유저의 포인트 조회 기능을 구현한다.
- 포인트 충전
    - 포인트 충전 기능을 구현한다.
    - 포인트는 0원에서 시작해서, 입력으로 받은 point만큼 증가시킨다.
    - 포인트는 최대 100,000 포인트까지 충전 가능하다.
    - 포인트 충전 시, 음수는 허용하지 않는다.
    - 포인트 충전 내역을 히스토리에 저장한다.
- 포인트 사용
    - 포인트 사용 기능을 구현한다.
    - 입력으로 받은 point 만큼 감소시킨다.
    - 보유하고 있는 포인트보다 많은 양의 포인트는 사용할 수 없다.
    - 포인트 사용 내역을 히스토리에 저장한다.
- 포인트 충전/사용 내역 조회
    - 특정 유저의 포인트 충전/이용 내역을 조회하는 기능을 구현한다.
    - 트랜잭션 타입(`CHARGE`, `USE`)으로 충전과 사용 내역을 구분한다.

---
## 테스트 전략

- 서비스 계층: 실제 객체 기반의 단위 테스트
- 도메인 로직: 예외 및 경계값 검증 테스트 포함
- 인수 테스트: RestAssured를 이용한 컨트롤러 통합 검증
- 예외 케이스: 음수 입력, 최대 잔고 초과, 잔고 부족 등
